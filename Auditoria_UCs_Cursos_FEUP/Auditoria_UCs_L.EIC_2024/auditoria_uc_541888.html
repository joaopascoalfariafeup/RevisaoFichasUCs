Laboratório de Bases de Dados e Aplicações Web

Apreciação da Ficha da Unidade Curricular

Aspeto a Avaliar Apreciação Qualitativa Apreciação Quantitativa (0–3) (a)
Objetivos de aprendizagem Objetivos bem estruturados, específicos e detalhados, com verbos de ação adequados e alinhamento claro com competências práticas 3
Tópicos programáticos Programa coerente e abrangente, mas sem mapeamento explícito para os objetivos de aprendizagem 2
Metodologias de ensino e aprendizagem Metodologia adequada ao caráter prático da UC, com acompanhamento tutorial, mas descrição pouco detalhada 2
Metodologias de avaliação Sistema de avaliação bem estruturado e detalhado, com componentes diversificadas e critérios claros de aprovação 3
Apreciação global Ficha bem estruturada com objetivos claros e avaliação detalhada, mas com oportunidades de melhoria no mapeamento 2.50

(a) Escala: 0 - Ausente ou totalmente inadequado; 1 - Parcialmente adequado; 2 - Largamente adequado; 3 - Totalmente adequado.

Pontos fortes

  • Objetivos de aprendizagem muito bem formulados, específicos e detalhados, com verbos de ação adequados (especificar, desenhar, obter, implementar, assegurar) que refletem diferentes níveis cognitivos
  • Sistema de avaliação muito bem estruturado e detalhado, com componentes diversificadas (projeto com 4 fases + exame) e critérios claros de aprovação
  • Definição clara das condições de frequência, incluindo participação obrigatória em todas as fases do projeto
  • Metodologia de ensino adequada ao caráter prático da UC, com acompanhamento tutorial durante as aulas laboratoriais
  • Programa abrangente que cobre adequadamente o desenvolvimento completo de aplicações web com bases de dados
  • Mecanismos de diferenciação individual dentro dos grupos de trabalho bem definidos
  • Coerência entre as versões portuguesa e inglesa da ficha

Pontos fracos

  • Ausência de mapeamento explícito entre os tópicos programáticos e os objetivos de aprendizagem
  • Falta de mapeamento explícito entre as componentes de avaliação e os objetivos de aprendizagem específicos
  • Descrição das metodologias de ensino pouco detalhada, não especificando atividades concretas de aprendizagem
  • Peso muito elevado do projeto (80%) pode criar pressão excessiva numa única componente
  • Impossibilidade de melhoria da classificação do projeto pode penalizar estudantes

Sugestões de melhoria

  • Incluir mapeamento explícito dos tópicos programáticos para os objetivos de aprendizagem (ex: "T1. Desenvolvimento de aplicação web... (O1, O2, O8, O9)")
  • Adicionar mapeamento das componentes de avaliação para objetivos específicos (ex: "ER - Especificação de Requisitos (O1); EBD - Especificação da Base de Dados (O3, O4, O5)")
  • Detalhar mais as metodologias de ensino, especificando atividades concretas como workshops tecnológicos, sessões de code review, ou apresentações intermédias
  • Considerar reequilibrar ligeiramente os pesos (ex: 75% projeto, 25% exame) para reduzir a dependência excessiva numa componente
  • Permitir melhoria limitada da classificação do projeto (ex: apenas para a componente PA - Produto e Apresentação) em casos excecionais
  • Especificar mais claramente como é aferida a "participação em cada componente do projeto" para efeitos de frequência
