Armazéns de Dados

Apreciação da Ficha da Unidade Curricular

Aspeto a Avaliar Apreciação Qualitativa Apreciação Quantitativa (0–3) (a)
Objetivos de aprendizagem Objetivos bem definidos, específicos e verificáveis, cobrindo adequadamente o ciclo completo de desenvolvimento de armazéns de dados 3
Tópicos programáticos Programa abrangente e bem estruturado, cobrindo todos os aspetos essenciais dos armazéns de dados de forma lógica e progressiva 3
Metodologias de ensino e aprendizagem Abordagem teórico-prática adequada com uso de laboratório, mas sem mapeamento explícito para os objetivos de aprendizagem 2
Metodologias de avaliação Avaliação equilibrada entre componente prática e teórica, mas sem mapeamento explícito dos elementos para os objetivos 2
Apreciação global Ficha bem estruturada com objetivos claros e programa sólido, mas com oportunidades de melhoria no alinhamento pedagógico 2.50

(a) Escala: 0 - Ausente ou totalmente inadequado; 1 - Parcialmente adequado; 2 - Largamente adequado; 3 - Totalmente adequado.

Pontos fortes

  • Objetivos de aprendizagem muito bem definidos, específicos e verificáveis, cobrindo todo o ciclo de vida de um projeto de armazém de dados
  • Programa abrangente e bem estruturado, seguindo uma progressão lógica desde conceitos básicos até implementação prática
  • Metodologia de ensino adequada ao caráter prático da unidade curricular, com aulas em laboratório que combinam teoria e prática
  • Avaliação equilibrada entre componente prática (50%) e teórica (50%), adequada aos objetivos aplicados da UC
  • Trabalho prático realista que percorre as diversas etapas de um projeto real de armazém de dados
  • Inclusão de apresentação oral no trabalho prático, simulando contextos profissionais reais
  • Carga de trabalho coerente com os 6 ECTS atribuídos (162 horas totais)
  • Coerência entre as versões portuguesa e inglesa da ficha

Pontos fracos

  • Ausência de mapeamento explícito entre os tópicos programáticos e os objetivos de aprendizagem
  • Falta de mapeamento explícito entre as componentes de avaliação e os objetivos de aprendizagem específicos
  • Descrição das metodologias de ensino poderia ser mais detalhada sobre como cada tipo de atividade contribui para objetivos específicos
  • Não são especificadas condições de assiduidade para obtenção de frequência
  • Critérios de avaliação do trabalho prático não são detalhados

Sugestões de melhoria

  • Incluir mapeamento explícito entre cada tópico programático e os objetivos de aprendizagem correspondentes (ex: "Modelação conceptual do armazém de dados: o modelo MultiDim (O3)")
  • Especificar claramente quais objetivos de aprendizagem são avaliados por cada componente (teste vs. trabalho prático)
  • Detalhar os critérios de avaliação do trabalho prático, incluindo aspetos como qualidade do modelo dimensional, implementação ETL, relatório e apresentação oral
  • Clarificar as condições de assiduidade necessárias para obtenção de frequência, se aplicáveis
  • Considerar incluir uma componente menor de participação presencial (5-10%) para valorizar o envolvimento nas aulas práticas
  • Especificar se existe nota mínima para o trabalho prático, além da nota mínima já definida para o teste
  • Detalhar melhor como será definido o "novo trabalho" para melhoria de classificação da componente prática
